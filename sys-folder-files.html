폴더

프로젝트 진행 시 사용하는 폴더 구조입니다. 브랜딩과 디자인 시스템을 적용한 서비스의 경우 작업 파일에 많은 디자이너와 개발자가 접근하기 떄문에 규칙을 짜 구성원들과 공유해 혼란을 최소화합니다. 각 폴더 내부는 원본(Resources) 재료(Assets) 작업 (.sketch, .psd 등) 결과물 (_export)로 이뤄집니다.

상위 폴더

Documentation 프로젝트에 연관된 모든 종류의 문서 (word,ppt,hwp)
External Works 프로젝트 내 리소스를 활용한 외부 업무 (마케팅, 홍보, 인쇄용 파일)
Repository-OS 플랫폼별 디자인 및 코드
Library 디자인 가이드라인 및 라이브러리
Resources 문서와 디자인에 사용될 수 있는 모든 외부 소스

하위 폴더

_Asstes Working Files에서 사용하는 요소. img, icon, font, graph 등
_Exports 해당 폴더에서 외부로 노출하는 결과물.
_Resource 해당 폴더 내에서만 필요한 리소스. 문서, 이미지, 레퍼런스 등
Working Files 작업 파일

이름

버전 관리 프로그램이나 Google Docs를 이용해 식별할 수 있는 유니크 이름만 작성해도 상관 없지만 로컬 파일이 필요할 경우 규칙을 정해 이름을 작성하는 것이 좋습니다. 폴더의 경우 대문자+띄어쓰기를 사용하며 파일은 소문자+대쉬를 사용합니다.
